# UC-1.5 通知中心

| 項目 | 內容 |
|------|------|
| **填表人** | 李芝瑢 |
| **商業流程編號** | UC-1.5 |
| **行為者** | 會員 |
| **內容概述** | 顯示系統通知,包含社交互動(留言、按讚、追蹤、互相追蹤)、系統公告、日記相關通知等,區別於私訊功能。 |
| **先決條件** | 使用者已登入。 |
| **後置條件** | 系統更新並顯示最新通知,已讀通知標記為 is_read = true。 |
| **主要流程** | 1. 使用者點擊通知圖示或進入通知中心頁面。<br>2. 系統從 notifications 資料表撈取該用戶的通知(GET /api/v1/notifications)。<br>3. 系統顯示未讀數量(badge)與通知列表,依時間降序排列。<br>4. 使用者點擊通知項目,系統標記為已讀並導航至相關頁面(如日記、評論等)。 |
| **替代流程** | a. 若無通知  顯示「目前無新通知」空狀態頁面。<br>b. 若點擊「全部標示為已讀」  系統批次更新所有未讀通知狀態。<br>c. 若點擊「清除已讀通知」  系統刪除所有已讀通知記錄。 |
| **輔助說明** | 通知類型包含:留言(comment)、按讚(like)、追蹤(follow)、互相追蹤(mutual_follow)、系統公告(announcement)。支援即時推播(WebSocket)或定時輪詢。通知中心與私訊(messages)功能獨立分開。 |
| **Screen 畫面** | 通知中心頁、頂部導航欄通知圖示 |
