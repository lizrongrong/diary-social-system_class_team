# UC-4.4 追蹤/取消追蹤他人

| 項目 | 內容 |
|------|------|
| **填表人** | 李芝瑢 |
| **商業流程編號** | UC-4.4 |
| **行為者** | 會員 |
| **內容概述** | 會員在瀏覽其他用戶個人頁或日記時,可快速追蹤或取消追蹤該用戶,建立單向社交關係。 |
| **先決條件** | 使用者已登入,且正在瀏覽其他用戶頁面。 |
| **後置條件** | 系統更新追蹤關係,追蹤數量即時更新,被追蹤者收到通知(若為新追蹤或互相追蹤)。 |
| **主要流程** | 1. 使用者在他人個人頁或日記頁點擊「追蹤」按鈕。<br>2. 系統立即建立追蹤關係(POST /api/v1/friends)。<br>3. 按鈕狀態變更為「已追蹤」,追蹤數量 +1。<br>4. 系統檢查是否為互相追蹤,若是則發送特殊通知。<br>5. 被追蹤者收到系統通知。 |
| **替代流程** | a. 若已追蹤該用戶  按鈕顯示「已追蹤」,點擊後取消追蹤  追蹤數量 -1  不發送通知。<br>b. 若對方已封鎖使用者  顯示「無法追蹤此用戶」錯誤訊息。 |
| **輔助說明** | 追蹤後可在「好友日記區」查看對方公開日記。追蹤關係為單向,不需對方同意。支援即時更新追蹤數量。 |
| **Screen 畫面** | 使用者個人頁、日記詳情頁、好友日記區 |
