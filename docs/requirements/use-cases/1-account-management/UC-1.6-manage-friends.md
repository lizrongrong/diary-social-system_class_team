# UC-1.6 管理好友名單

| 項目 | 內容 |
|------|------|
| **填表人** | 李芝瑢 |
| **商業流程編號** | UC-1.6 |
| **行為者** | 會員 |
| **內容概述** | 會員可查看、搜尋、追蹤或移除好友,採用單向追蹤機制(類似 Instagram/Twitter),無需對方確認即可追蹤。 |
| **先決條件** | 使用者已登入。 |
| **後置條件** | 好友名單更新成功,系統記錄追蹤關係至 friends 資料表。 |
| **主要流程** | 1. 使用者進入好友管理頁面。<br>2. 使用者可搜尋其他用戶(依 username 或 display_name)。<br>3. 使用者點擊「追蹤」按鈕。<br>4. 系統立即建立追蹤關係,無需對方確認。<br>5. 若雙方互相追蹤,系統發送「互相追蹤」通知給對方。<br>6. 使用者可查看「追蹤中」(following)和「追蹤者」(followers)列表。 |
| **替代流程** | a. 若搜尋不到用戶  顯示「查無此用戶」提示。<br>b. 若已追蹤該用戶  顯示「已追蹤」狀態,可點擊「取消追蹤」。<br>c. 若嘗試追蹤自己  系統顯示錯誤訊息「無法追蹤自己」。 |
| **輔助說明** | 採用單向追蹤機制,追蹤後可立即查看對方的公開日記。系統會檢測互相追蹤狀態並發送通知。未來可擴展封鎖功能。 |
| **Screen 畫面** | 好友管理頁、用戶搜尋頁、用戶個人頁 |
