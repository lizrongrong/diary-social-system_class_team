# UC-2.8 日記情緒標籤設定

| 項目 | 內容 |
|------|------|
| **填表人** | 李芝瑢 |
| **商業流程編號** | UC-2.8 |
| **行為者** | 會員 |
| **內容概述** | 使用者在新增或編輯日記時,可選擇預設的情緒標籤(如開心、難過、焦慮等),記錄當下心情,便於情緒追蹤與分析。此功能為 UC-2.1 與 UC-2.2 的子流程。 |
| **先決條件** | 使用者已登入帳號,正在新增或編輯日記。 |
| **後置條件** | 情緒標籤記錄至 diary_tags 資料表(tag_type = 'emotion'),可用於情緒統計與 AI 分析。 |
| **主要流程** | 1. 使用者在日記編輯介面的「情緒」區域點擊情緒選擇器。<br>2. 系統顯示預設情緒標籤列表(從 emotion_tags 表讀取),每個標籤包含名稱、圖示(emoji)與顏色。<br>3. 使用者可選擇一個或多個情緒標籤(如「開心 😊」、「焦慮 😰」)。<br>4. 系統即時顯示已選情緒,使用者可取消選擇。<br>5. 使用者儲存或發布日記時,系統將選中的情緒插入 diary_tags 表:<br>&nbsp;&nbsp;&nbsp;- tag_type = 'emotion'<br>&nbsp;&nbsp;&nbsp;- tag_value = 情緒名稱(如「開心」、「焦慮」)<br>&nbsp;&nbsp;&nbsp;- diary_id = 當前日記 ID |
| **替代流程** | a. 若使用者未選擇情緒標籤  系統可選擇性觸發 AI 自動辨識情緒(基於 UC-2.10 分析結果),並推薦情緒標籤供使用者確認。<br>b. 若 emotion_tags 表為空  系統使用預設情緒列表(開心、難過、生氣、焦慮、平靜、興奮)。<br>c. 若選擇超過 5 個情緒  系統提示「最多選擇 5 個情緒」限制。 |
| **輔助說明** | 情緒標籤來源為 emotion_tags 資料表,管理員可透過後台新增/編輯情緒選項(emotion_name, emotion_icon, color_code)。支援 AI 自動分析情緒建議功能(整合 UC-2.10)。情緒資料可用於月度回顧(UC-2.11)、情緒趨勢圖表等進階功能。 |
| **Screen 畫面** | 日記編輯頁(新增/編輯日記)、情緒選擇器彈窗 |
